Architecture

Data Flow — GraphQL + Code Generation

Define operations src/graphql/operation/*.graphql
↓
pnpm codegen GraphQL Code Generator → types + React hooks
↓
Generated hooks useProductsQuery, useAddToCartMutation, etc.
↓
Components Client components use hooks, Server components use Apollo client directly

Request Flow — App Router + i18n + Multi-Store

Incoming Request us.myapp.com/products or eu.myapp.com/fr/products
↓
proxy.ts Resolve store from hostname → detect/redirect locale → auth check
↓
[locale]/layout.tsx Load translations, set lang/dir, init store config + currency
↓
Route Group Layout (auth) | (shop) | (dashboard) — session check
↓
Server Component Fetches data via Apollo Client with store/locale headers
↓
Client Components Locale-aware UI, currency-formatted prices, RTL support

Multi-Store Resolution

us.myapp.com → US store — English, USD
eu.myapp.com → EU store — English/French/German, EUR/GBP
me.myapp.com → Middle East store — Arabic (RTL)/English, AED/USD
in.myapp.com → India store — English, INR/USD
