Architecture Principles

1. Feature-Sliced Design

Each feature module is self-contained with its own components, hooks, services, utils, types, and tests. This enables independent development and clean code-splitting at feature boundaries.

2. Co-located Testing (Hybrid)

Unit tests live beside source files — impossible to miss. __tests__/ holds only shared setup (MSW handlers, canvas mocks, fixtures) and Playwright E2E specs.

3. Data Flow

User Interaction
↓
