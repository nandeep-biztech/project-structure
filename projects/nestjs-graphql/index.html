Co-located Module Tests — Each Feature Has Its Own __tests__/ Directory

Unit tests live in __tests__/ within each feature module. E2E tests live in test/e2e/. The rule: unit tests mock everything at the boundary, E2E tests mock nothing except external services (Stripe, SendGrid, etc.).

Architecture Principles

1. Feature Module Architecture

Each module is self-contained with its own resolver, service, models, DTOs, entities, loaders, mappers, policies, and tests. Modules communicate through exported services and loaders — never by reaching into another module's internals.

2. Separation of Concerns — models/ vs entities/

models/ holds GraphQL @ObjectType() classes only. entities/ holds TypeORM @Entity() classes only. A mapper/ converts between the two — so a schema change never requires a DB migration and vice versa.

3. Request Flow

GraphQL Request
↓
Apollo Server → Complexity + Depth plugins
↓
Auth Guard JWT validation → @CurrentUser()
↓
Resolver @Query / @Mutation / @ResolveField
↓
Service Business logic + CASL authorization
↓
Repository TypeORM query → PostgreSQL

4. Apollo Plugin Pipeline

ComplexityPlugin Rejects over-budget queries
↓
DepthLimitPlugin Max nesting depth (default: 7)
↓
DataloaderPlugin Per-request DataLoader injection
↓
LoggingPlugin Op name + duration + error status
↓
TracingPlugin OpenTelemetry span per operation
↓
SentryPlugin Captures resolver errors with context
↓
PersistedQueryPlugin APQ — caches query documents by hash
